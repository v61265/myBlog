什麼是物件導向

從剛剛的例子開始說吧：

1function createWallet(init) {
2    var money = init
3    return {
4        add: function(num) {
5            money += num
6        },
7        deduct: function(num) {
8            money -= num
9        }
10    }, getMoney() {
11        return money
12    }
13}
14

15var myWallet = createWallet(99)
16myWallet.add(1)
17myWallet.deduct(10)
18console.log(myWallet.getMoney()) // 90

這個例子中回傳的值是一個物件，其實就算是物件導向。在使用 JS 時，也時常不是直接 call 一個 function ，而是對某個物件做操作，這種做法的好處是方便模組化。

class

從 ES6 的 class 開始談起。

首先， class 的名稱一定是大寫開頭，例如：

1class Dog {
2  sayHello() {
3    console.log("hello");
4  }
5}

class 有點像設計圖，當我們實際使用前時，要用 new 將 class 實體化 (instance)：

1var d = new Dog();
2d.sayHello(); // hello

另一個概念是 this ，它會指向呼叫它的東西：

1class Dog {
2  setName(name) {
3    this.name = name;
4  }
5  getName() {
6    return this.name;
7  }
8}
9

10var d = new Dog();
11d.setName("jojo");
12console.log(getName()); // jojo

上面範例中 d.setName('jojo') 中的 this 因為是由 d 呼喚的，因此 this 當然就指向變數 d 。 class 中 setName(name) 這樣的函數被稱為 setter ，讓裡面存取到外面的值；而 sayHello() 則叫 getter ，是讓外面得到 class 的值。 另外我們也可以直接這樣寫：

1d.name = "dio";
2console.log(d.name); // dio

但還是建議用 setter 和 getter 。

如果想要用像是函式傳參數的方式設定，可以用 建構子 constructor：

1class Dog {
2  constructor(name) {
3    this.name = name;
4  }
5  getName() {
6    return this.name;
7  }
8}
9

10var d = new Dog("jojo"); // 字串 'jojo' 被傳入 constructor() 中
11console.log(getName()); // jojo
12

13var d = new Dog("dio");
14console.log(getName()); // dio

ES5 的 class

在 ES5 中沒有 class ，因此要這樣寫：

1function Dog(name) {
2    var myName = name
3    return {
4        getName: function() {
5            return myName
6        },
7        sayHello: funcrion() {
8            console.log(myName)
9        }
10    }
11}
12

13var d = Dog('jojo')
14d.sayHello // jojo
15

16var b = Dog('dio')
17d.sayHello // dio

不過因為每次都是呼叫一個新的物件，會出現這種狀況：

1console.log(b.sayHello === d.sayHello); // false

不過兩個是同個 function ，所以共用同個 function 比較省記憶體吧？

因此在 ES5 中，可以將 function 當作 constructor 用：

1function Dog(name) {
2  this.name = name;
3}
4

5var d = new Dog("abc");
6console.log(d); // Dog { name: 'abc' }

自動變成物件了！ 不過這樣要怎麼知道是 constructor 還是平常的 function？只有加 new 才會被認定是 constructor ，如果沒加就是 function 。

設定屬性的問題搞定了，但要怎麼設定輸出名字和其他操作ㄋ？這時候可以把東西掛在 .prototype 上：

1Dog.prototype.sayHello = function () {
2  console.log(this.name);
3};
4

5var d = new Dog("jojo");
6d.sayHello; // jojo

Prototype

JavaScript 中，每個變數都有個隱藏屬性 __proto__ ，暗示如果在 d 上面找不到 sayHello 的屬性：

1function Dog(name) {
2  this.name = name;
3}
4

5Dog.prototype.sayHello = function () {
6  console.log(this.name);
7};
8

9var d = new Dog("jojo");
10d.sayHello; // jojo
11

12console.log(d.__proto__);
13// Dog { sayHello: [Function (anonymous)] }
14// 其實就是 Dog.prototype

當我們呼叫 d.sayHello 時，我們其實是做了：

  1. d 本身是否有 sayHello
  2. d.__proto__ 是否有 sayHello ，也就是 Dog.prototype
  3. 沒有的話就找 d.__proto__.__proto__ ，也就是 Object.prototype
  4. 如果還是沒有就找 d.__proto__.__proto__.__proto__ ，沒有的話會回傳 null。
  5. null 代表找到頂了，沒有的話就會拋出錯誤。

以上都是只要有就會回傳值，沒有的話才往下進行，這個步驟被稱為原型練（Prototype Chain）。

我們來看一下：

1console.log(d.__proto__);
2// Dog.prototype
3// 結果：Dog { sayHello: [Function (anonymous)] }
4console.log(d.__proto__.__proto__);
5// Dog.prototype.__proto__
6// Object.prototype
7// 結果：{}
8console.log(d.__proto__.__proto__.__proto__);
9// null

他們之間的關係如下：

1d.__proto__ = Dog.prototype;
2d.__proto__.__proto__ = Object.prototype;
3Dog.prototype.__proto__ = Object.prototype;

因此我們也可以設定 Object 的 prototype ，這樣就會在第三個步驟呼叫到結果：

1Object.prototype.sayHello = function () {
2  console.log("object", this.name);
3};
4

5var d = new Dog("jojo");
6d.sayHello; // object jojo

如果同時設定 Object 和 Dog 的 prototype ，則會因為原型鍊會先選到 Dog 的：

1Dog.prototype.sayHello = function () {
2  console.log(this.name);
3};
4

5Object.prototype.sayHello = function () {
6  console.log("object", this.name);
7};
8

9var d = new Dog("jojo");
10d.sayHello; // jojo

同理，此處的 Object 如果被換成 Function ，第四個步驟就會被換成 Function.prototype 。

new 到底做了什麼

function.call() 這個函數可以指定 function 中的 this 值：

1function test() {
2  console.log(this);
3}
4

5test.call(123); // [Number: 123]

接著來拆解 new 到底幫我們做了甚麼，因此用另一個 function 來模擬：

1function newDog(name) {
2  // 模擬 new 做了一些事情
3}
4

5// 最後目標
6var a = newDog("jojo");
7a.sayHello(); // 印出 jojo
  1. 建立一個 object，並將值傳入
1function newDog(name) {
2  var obj = {};
3  Dog.call(obj, name); // 第一個是 this ，後面依序是傳入值
4  console.log(obj);
5}
6

7var a = newDog("jojo"); //{ name: 'jojo' }
  1. 設定 prototype 連結
1function newDog(name) {
2  var obj = {};
3  Dog.call(obj, name); // 第一個是 this ，後面依序是傳入值
4  obj.__proto__ = Dog.prototype;
5}
6

7var a = newDog("jojo");
  1. 回傳 object
1function newDog(name) {
2  var obj = {};
3  Dog.call(obj, name); // 第一個是 this ，後面依序是傳入值
4  obj.__proto__ = Dog.prototype;
5  return obj;
6}
7

8var a = newDog("jojo");
9a.sayHello(); // 印出 jojo

就完成ㄌ！

Inheritance

設想有一個狗的 class ，今天我需要設定黑狗和白狗，這時有名字、會叫、丟飛盤會去接回來之類的和狗有關的屬性就不用再設定一次了。要是有人問你「黑狗有幾個眼睛」時，只要回頭查看「狗」的條目就可以了。這就是 Inheritance 繼承的概念。

ES6 中的繼承可以這樣寫：

1class BlackDog extands Dog{
2  // 其他黑狗的屬性
3}
4

5const d = BlackDog('jojo')
6d.sayHello()

上面的例子中 d.sayHello() 實際上是往上找到 Dog 的屬性。

此時若我們想讓黑狗被建立的時候就呼叫 sayHello()：

1class BlackDog extands Dog{
2    constructure() {
3        this.sayHello()
4    }
5}
6

7const d = BlackDog('jojo')

這樣會噴錯，因為在 constructor 中呼叫 this 前要用 super() 另外引入上一層的constructor ，如下：

1class BlackDog extands Dog{
2    constructure(name) {
3        super(name)
4        this.sayHello()
5    }
6}
7

8const d = BlackDog('jojo') // jojo

this

this 在物件導向中被使用，可以用代表其所對應到的 instance 。

如果直接呼叫 this 例如：

1function test() {
2  console.log(this);
3}
4

5test();

會出現一長串的東西。

若不是物件導向的環境下，預設值為 Global ，node.js 跑是 global 的變數，瀏覽器則是 window 。也可以在檔案最上方輸入 'use strict'; 進入嚴格模式，此時的預設值就會是 undefined 。

另一個例外是使用 DOM 的時候：

1document.querySelector(".dom").addEventListener("click", function () {
2  console.log(this); // 點擊到的東西
3});

call 和 apply

.call() 的第一個值被預設為 this 的值：

1function test() {
2  console.log(this);
3}
4

5test.call(123); // [Number: 123]

apply 也是：

1function test() {
2  console.log(this);
3}
4

5test.apply(123); // [Number: 123]

兩個的差別是後面的參數引入的方法， call 就是用逗號連接，但 apply 只有兩個參數，第二個參數則是將要傳入的參數們用陣列包起來。

怎麼看 this

1const obj = {
2  a: 123,
3  test: function () {
4    console.log(this);
5  },
6};
7

8obj.test(); // this 對應到 obj 本身

this 和放在哪裡無關，而是看呼叫的方法。例如以下寫法雖然一樣，結果卻不同：

1const obj = {
2  a: 123,
3  test: function () {
4    console.log(this);
5  },
6};
7

8const func = obj.test;
9func(); // undefined

因為第一個寫法 obj.test() 可被視為 obj.test.call(obj) ，因此會呼叫到 obj 。

bind

小小練習，自己先猜猜看答案：

1function log() {
2  console.log(this);
3}
4

5var a = { a: 1, log: log };
6var b = { a: 2, log: log };
7

8log(); // global
9a.log(); // a
10

11b.log.apply(a); // a，因為 call 的值優先

如果希望不管怎麼呼叫， this 的值都不會變，可以是用 .bind() ：

1const bindTest = obj.test.bind(obj);

之後不管從哪裡呼叫 bindTest() ， this 的結果都是 obj 。

bind 和 call / apply 的差別在於，前者會回傳一個新的 function ，後者則是直接呼叫。

碰到箭頭函式，一切都不一樣ㄌ

使用到箭頭函式時， this 的值和如何呼叫沒有關係，此時的規則和 scope 比較像，也就是和定義在哪裡有關係。

可以看這個例子：

1class Test {
2  run() {
3    consoel.log(this); // Test
4    setTimeOut(function () {
5      console.log(this); // unefined
6    }, 1000);
7  }
8}
9

10const t = newTest();
11t.run();

但如果用箭頭函式：

1class Test {
2  run() {
3    consoel.log(this); // Test
4    setTimeOut(() => {
5      console.log(this); // Test
6    }, 1000);
7  }
8}
9

10const t = newTest();
11t.run();
